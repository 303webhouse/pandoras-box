# P2.0 — Options Pricing Fix (Push expiry/option_type Filters to UW API)

**Type:** Hotfix (production regression)
**Severity:** P0 — 12 of 15 open positions show null current_price and $0 unrealized P&L
**Priority:** Ship before pre-market tomorrow
**Estimated effort:** ~5 minutes (one function signature change in uw_api.py)

---

## Context

After the Polygon → UW migration, mark-to-market for option positions stopped working. Today's `/api/v2/positions` response shows current_price=null and unrealized_pnl=0 for every option position the user holds (SMH, URA, IWM, BX, RSP, COUR, DINO, TSLA, XLY, HYG, DBA, PLTR — 12 positions total). The 3 equity positions (SOXS, SRTY, TSLQ) price correctly because they go through `get_snapshot`, not the options pricer.

This was masked through the P1.x heatmap fixes because they fixed a different code path. Options pricing has been broken since the UW migration.

### Root cause (proven empirically tonight)

`backend/integrations/uw_api.py` line 450:
```python
data = await _uw_request(f"/api/stock/{underlying.upper()}/option-contracts")
```

This call passes NO query parameters. Per UW's OpenAPI spec (`api_spec.yaml` line 9315):

> "The data returned by /stock/:ticker/option-contracts has been limited to 500 results"

So calling the bare path returns ALL contracts capped at 500, spread across all expirations. For popular tickers with weeklies + monthlies + LEAPS, 500 isn't enough to cover everything, and contracts the position pricer needs get cut off.

### Empirical proof (PowerShell against the live UW API tonight)

**Bare call** `/api/stock/SMH/option-contracts`:
- 500 contracts returned (cap hit)
- 19 unique expirations represented (5/1, 5/8, 5/15, 5/22, 5/29, 6/5, 6/18, 7/17, 8/21, 9/18, ...)
- Only 70 contracts for the 5/15 expiry the SMH spread position needs

**Filtered call** `/api/stock/SMH/option-contracts?expiry=2026-05-15`:
- 218 contracts returned (no cap because 5/15 only has 218 strikes×types total)
- BOTH the 450 strike put (long leg) AND 440 strike put (short leg) present:
  - SMH260515P00450000: nbbo_bid 4.40, nbbo_ask 5.10, last_price 4.75
  - SMH260515P00440000: nbbo_bid 3.00, nbbo_ask 3.70, last_price 3.70
- Computed spread mid: 4.75 - 3.35 ≈ $1.40 (entry was $1.38 → ~breakeven)

The contracts exist. The endpoint works. We just need to ASK for the expiry we want.

### Why client-side filtering doesn't save us

The existing code does post-filter:
```python
if expiration_date and c_expiry != str(expiration_date)[:10]:
    continue
```
But this filter runs AFTER the 500-cap has already been applied at the UW server. By the time client-side filtering runs, the 5/15 contracts we need may already be gone.

### Why `_uw_request` already supports this

`_uw_request(path, params=None)` at line 115 already accepts query params. Other callers (e.g., `get_news_headlines` line 768, `get_congress_recent_trades` line 807) already pass params successfully. We just need `get_options_snapshot` to do the same.

### Why this didn't burn budget visibly

UW didn't 404 on these calls — they returned 200 with 500 contracts. The pricer just couldn't find the strike/expiry combo it wanted in those 500, so `_find_contract` returned None, and the wider `try/except Exception` block swallowed the path silently. Logged at WARNING but no Discord alert, and `price_updated_at` stays at whatever it was previously (because the SUCCESS branch — which sets price_updated_at — never ran).

---

## Goal

In `get_options_snapshot()`, build a `params` dict from the existing function arguments and pass it to `_uw_request`. UW's spec supports `expiry` and `option_type` as native query params, so push those down server-side. Strike filters (strike_gte/strike_lte) are NOT supported as native UW params, so leave those as client-side post-filters.

## The Change

**File:** `backend/integrations/uw_api.py`

**Find (exact match — currently around lines 449–451 in `get_options_snapshot`):**

```python
    data = await _uw_request(f"/api/stock/{underlying.upper()}/option-contracts")
    if not data or "data" not in data:
        return None
```

**Replace with:**

```python
    # P2.0 fix 2026-04-28: UW limits this endpoint to 500 results per call.
    # For tickers with many expirations (SMH, SPY, IWM, TSLA, etc.) the 500-cap
    # cuts off the specific expiry/strike combos the position pricer needs,
    # producing null current_price for every option position. Push native
    # expiry/option_type filters down to UW so the response stays under cap and
    # always contains the strikes we need. Strike range filters (strike_gte/lte)
    # are not native UW params and remain client-side post-filters below.
    params: Dict[str, Any] = {}
    if expiration_date:
        params["expiry"] = str(expiration_date)[:10]
    if contract_type:
        params["option_type"] = contract_type

    data = await _uw_request(
        f"/api/stock/{underlying.upper()}/option-contracts",
        params=params or None,
    )
    if not data or "data" not in data:
        return None
```

That's the only change. Do not modify the cache_get/cache_set logic above and below this block. Do not touch `_parse_option_symbol`, `_find_contract`, `_get_contract_mid`, or any of the spread/single-leg/multi-leg helpers.
