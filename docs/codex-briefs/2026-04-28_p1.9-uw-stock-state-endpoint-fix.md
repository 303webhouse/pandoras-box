# P1.9 — UW Snapshot Endpoint Path Fix (404 Root Cause)

**Type:** Hotfix (production incident — actual root cause of today's breaker storm)
**Severity:** P0 — every quote fetch returns 404, breaker re-trips every cooldown
**Priority:** Ship before pre-market tomorrow
**Estimated effort:** 2 minutes (1-line code change + commit + push)

---

## Context

P1.8 fixed the *recovery* bug in the circuit breaker. After deploying P1.8, the breaker still trips. Fresh logs reveal the actual root cause:

```
ERROR:uw_api:UW API /api/stock/SPY/state: HTTP 404 — {"error":"Route not found",
  "help":"If you are using an LLM/AI or if you are a coding agent see
  https://unusualwhales.com/information/how-to-fix-404-errors-when-using-ai-tools..."}
```

UW serves a 404 with a help page literally about AI tools hitting hallucinated endpoints. The endpoint `/api/stock/{ticker}/state` does not exist. The actual endpoint per UW's OpenAPI spec is `/api/stock/{ticker}/stock-state`.

This was introduced in commit `6262f54` (P1.5: replace yfinance with UW /state in get_snapshot). The migration used the wrong path name. UW's MCP server tool is named `stock_state`, which apparently led the prior agent to assume the URL was `/state` rather than `/stock-state`.

### Verification

The repo contains `api_spec.yaml` (UW's full OpenAPI spec, 17,592 lines, downloaded from `https://api.unusualwhales.com/api/openapi`). An audit script (see `C:\temp\audit_uw_paths.py`) cross-referenced all 21 `_uw_request()` calls in `uw_api.py` against the spec. Result: exactly one phantom endpoint — `/api/stock/{ticker}/state`. All 20 others (`/info`, `/flow-recent`, `/max-pain`, `/iv-rank`, `/greek-exposure`, etc.) match the spec exactly.

UW MCP test confirms `stock-state` returns the expected data shape (close, prev_close, open, high, low, volume, total_volume, prev_close, market_time, tape_time) — the exact fields `get_snapshot()` parses.

### Why it didn't blow up the WHOLE app

`/api/stock/{ticker}/info` is correct (matches spec) and is called separately with a 24h cache, so static metadata still works. Only the *live* OHLCV path was broken — which is why the heatmap showed data but it was *stale* (falling back to whatever `get_snapshot()`'s caller does on `None`).

## The Change

**File:** `backend/integrations/uw_api.py`

**Find (exact match — currently in `get_snapshot()`):**

```python
    # State is the critical path — direct call, no fallback acceptable
    state_resp = await _uw_request(f"/api/stock/{ticker.upper()}/state")
```

**Replace with:**

```python
    # State is the critical path — direct call, no fallback acceptable
    # P1.9 fix 2026-04-28: endpoint is /stock-state not /state. Prior commit 6262f54
    # introduced the wrong path (UW MCP tool is named stock_state, but the REST URL
    # uses kebab-case). UW returned 404 on every call, tripping the circuit breaker
    # and degrading the heatmap to stale fallback data. Validated against api_spec.yaml.
    state_resp = await _uw_request(f"/api/stock/{ticker.upper()}/stock-state")
```

That's the only change. Do not touch anything else in the file. Do not modify the `/info` call (it's already correct).
