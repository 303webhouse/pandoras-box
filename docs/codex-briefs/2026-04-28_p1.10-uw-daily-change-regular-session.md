# P1.10 — UW Daily Change Calculation Fix (Regular-Session OHLC Bars)

**Type:** Hotfix (production incident — final root cause)
**Severity:** P0 — daily change %, prev close, and reference price are all wrong outside regular hours
**Priority:** Ship before pre-market tomorrow
**Estimated effort:** ~10 minutes (one new helper + one block change in `get_snapshot()`)

---

## Context

P1.9 fixed the URL path so the hub stops 404'ing. UW is now returning real data. But the *interpretation* of that data is wrong, producing impossible numbers on the heatmap.

**Real data for XLK on 2026-04-28** (verified via UW MCP `/api/stock/XLK/ohlc/1d`, cross-checked StockAnalysis.com):

| Source | Prev close | Today close | Change |
|---|---|---|---|
| StockAnalysis.com | $160.57 | $157.85 | **−1.69%** |
| UW OHLC bar (`market_time: r`, 2026-04-27) | — | **$160.57** | — |
| UW OHLC bar (`market_time: r`, 2026-04-28) | — | **$157.85** | — |

**What the hub was reporting (wrong):** XLK $158.58, **+0.46%**

### Why it's wrong

UW's `/api/stock/{ticker}/stock-state` endpoint returns DIFFERENT field meanings depending on the current market session:

| Session | What `prev_close` means | What `close` means |
|---|---|---|
| Pre-market | Yesterday's regular close | Latest pre-market trade |
| **Regular hours** | **Yesterday's regular close** | **Latest regular-session trade** |
| Post-market | **Today's regular close** | Latest post-market trade |
| Closed (overnight) | Today's regular close | Today's regular close |

So when `get_snapshot()` does `(close - prev_close) / prev_close`, the result is correct ONLY during regular hours. After 4 PM ET, `prev_close` rolls forward to today's regular close and `close` becomes the post-market drift, producing a meaningless "change vs today's regular close" number.

For XLK after the close on 2026-04-28: UW returned `prev_close: 157.85` and `close: 158.58`. That's `(158.58 - 157.85) / 157.85 = +0.46%` — which is post-market drift, not today's daily change of −1.69%.

### Verification that the OHLC endpoint has the right data

`/api/stock/{ticker}/ohlc/1d` returns one bar per market session per day, tagged with `market_time`:
- `pr` = pre-market session
- `r` = **regular session** (the one we want)
- `po` = post-market session

For each trading day, taking the bar where `market_time == "r"` gives the canonical "today's regular session" close and OHLC. The previous regular-session bar gives prev close. This is invariant across pre-market / regular / post-market / overnight — it's the same data regardless of when we query.

XLK 4/28 regular bar: `{ "open": "157.11", "high": "158.64", "low": "156.16", "close": "157.85", "date": "2026-04-28", "market_time": "r" }`. Previous regular bar (4/27): `close: "160.57"`. Computed change: (157.85 - 160.57) / 160.57 = **−1.69%**. Matches reality.

### Surface area audit (so this fix doesn't break anything else)

`grep` of the codebase shows `get_snapshot()`'s output is consumed via these fields:

| Field | Consumers | Current behavior | What it should be |
|---|---|---|---|
| `lastTrade.p` | hydra_squeeze, strc_monitor, sectors, macro_strip, context_modifier, signal_enricher, hermes, circuit_breaker | live last trade (correct in all sessions) | **unchanged** |
| `day.c` | sectors, context_modifier, circuit_breaker | live last trade | today's regular-session close |
| `prevDay.c` | sectors, macro_strip, trip_wire_monitor, ticker_profile, polygon_prefilter, signal_enricher, hermes, context_modifier | varies by session (BROKEN post-market) | yesterday's regular-session close |
| `todaysChange` / `todaysChangePerc` | uw_flow_poller filtering | varies by session (BROKEN post-market) | (today reg close − yest reg close) |

So `lastTrade.p` stays as the live tick (correct for "what does this trade at right now"). Everything tied to *daily change* needs to come from regular-session OHLC bars.

```python
async def _get_regular_session_change(ticker: str) -> Optional[Dict[str, Any]]:
    """Compute today's regular-session daily change from UW OHLC bars.

    P1.10 fix 2026-04-28: UW's /stock-state endpoint returns prev_close/close
    fields whose meaning shifts based on the current market session — during
    post-market, prev_close rolls forward to today's regular close and close
    becomes the post-market last trade, producing a bogus "+0.46%" reading
    when XLK was actually -1.69% on the day.

    /api/stock/{ticker}/ohlc/1d returns one bar per session per day tagged with
    market_time ('pr', 'r', 'po'). Taking the last two bars where market_time
    == 'r' gives a session-invariant daily change calculation that's correct
    pre-market, regular hours, post-market, and overnight.

    Returns dict with today_close, prev_close, change, change_pct, today_open,
    today_high, today_low, today_volume — or None if data unavailable. Cached
    under the same 'quote' TTL as get_snapshot.
    """
    cache_key = f"{ticker.upper()}:reg_change"
    cached = await cache_get("quote", cache_key)
    if cached is not None:
        return cached

    resp = await _uw_request(f"/api/stock/{ticker.upper()}/ohlc/1d")
    if not resp or "data" not in resp:
        return None

    bars = resp["data"]
    if not isinstance(bars, list):
        return None

    # Filter to regular-session bars only ('r'), preserve order
    regular_bars = [b for b in bars if b.get("market_time") == "r"]
    if len(regular_bars) < 2:
        logger.warning("UW ohlc/1d returned <2 regular-session bars for %s", ticker)
        return None

    def _f(v):
        try:
            return float(v) if v is not None else None
        except (ValueError, TypeError):
            return None

    today = regular_bars[-1]
    prev = regular_bars[-2]

    today_close = _f(today.get("close"))
    prev_close = _f(prev.get("close"))
    if today_close is None or prev_close is None or prev_close == 0:
        return None

    change = today_close - prev_close
    change_pct = (change / prev_close) * 100

    result = {
        "today_close": today_close,
        "today_open": _f(today.get("open")),
        "today_high": _f(today.get("high")),
        "today_low": _f(today.get("low")),
        "today_volume": today.get("total_volume") or today.get("volume"),
        "prev_close": prev_close,
        "change": change,
        "change_pct": change_pct,
    }
    await cache_set("quote", cache_key, result)
    return result


async def _get_info_cached_long(ticker: str) -> Dict[str, Any]:
```

That adds the helper. The existing `_get_info_cached_long` body stays exactly as it is.
