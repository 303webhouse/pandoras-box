## Why this exists (root cause — verified live)
`hub_get_flow_radar` returns `flow_data_available:false` / `$0` during market hours even though `uw:flow:{ticker}` keys are **full of real data in the wrong shape**.

**Cache-key collision.** `get_flow_per_expiry()` caches via `cache_set("flow", ticker)`; cache keys are `f"uw:{endpoint_type}:{key}"` → **`uw:flow:{ticker}`** — the exact key the committee readers expect to hold the canonical *summary* dict. So the UW response cache overwrites it with a **raw list of per-expiry rows**; every reader does `json.loads(raw).get(...)` → `AttributeError` on a list → degrades to no-flow. **Chunk 4 activated it** (poller calls `get_flow_per_expiry` for all 20 `LIQUID_UNIVERSE` tickers every 5 min). Live keys = exactly `LIQUID_UNIVERSE` (XLF absent) = poller's set, confirming the source. No new outage (committee was already flow-blind on empty keys), but it's the Chunk-4 follow-through.

### Phase-0 evidence (settled)
- `uw:flow:NVDA` live = JSON **list** (per-expiry shape); `hub_get_flow_radar(NVDA)` = `flow_data_available:false`.
- Cache builder `uw_api_cache.py:59/76` = `f"uw:{endpoint_type}:{key}"`; `CACHE_TTLS["flow"]=300`. `"flow"` namespace used **only** by `get_flow_per_expiry` (uw_api.py:829,839) — rename safe.
- bias_scheduler `_uw_flow_polling_loop` (start L2804) calls `get_flow_recent` (~800/session live) but writes nothing usable (no `uw:flow:XLF` despite XLF in its list). Its only state-write is the `uw:flow:` set — no bias-factor/other collateral (verified).
- Webhook writers (`flow_ingestion.py`, `api/uw.py`) not firing. `api/flow.py:458` writes only `if not existing` (manual fallback) — see §A-4.

### Canonical contract — the critical trap (verify VALUES, not just keys)
Summary dict at `uw:flow:{ticker}`. **Critical key: `sentiment` (NOT `flow_sentiment`).** Reader needs:

| Reader | keys |
|---|---|
| `api/flow_radar.py` | `sentiment, pc_ratio, total_premium, call_premium, put_premium, change_pct` |
| `scanners/cta_scanner.py` | `net_premium, sentiment, unusual_count` |
| `scoring/contrarian_qualifier.py` | `sentiment, net_premium` |
| `api/unified_positions.py` | `sentiment` (+ premium) |
| `scanners/hydra_squeeze.py` | `total_call_premium…/call_count…` — **pre-existing mismatch, BANKED (§7c)** |

**⚠️ A1 — sentiment SEMANTICS (deploy-blocker).** Same key/values, different logic:
- **Canonical** (bias_scheduler L2873): **premium-based** — `BULLISH if call_prem > put_prem*1.3`, `BEARISH if put_prem > call_prem*1.3`, else NEUTRAL.
- **Poller `flow_sentiment`** (uw_flow_poller L96-106): **volume-based** (pc_ratio 0.7/1.3) with a premium override only at 2×.
Reusing `row["flow_sentiment"]` would silently shift committee flow sentiment to a more volume-driven definition every reader was NOT built around — and no reader would catch it. **Decision (ATLAS): match canonical** — compute the summary's `sentiment` from call/put **premium at 1.3×**. Do NOT reuse `flow_sentiment`. (`flow_events`/Postgres keeps its own volume-based `flow_sentiment` — unchanged.)

```python
def build_flow_summary(row: dict) -> dict:
    """Canonical uw:flow:{ticker} summary for committee/flow_radar/cta/contrarian.
    sentiment is PREMIUM-based to match the legacy canonical writer (NOT row's
    volume-based flow_sentiment). Single source of the committee flow contract."""
    cp = row["call_premium"]; pp = row["put_premium"]
    if cp > pp * 1.3:
        sentiment = "BULLISH"
    elif pp > cp * 1.3:
        sentiment = "BEARISH"
    else:
        sentiment = "NEUTRAL"
    pc = row.get("pc_ratio")
    if pc == 999.0:                      # A3: all-put sentinel — don't leak the magic
        pc = None                        #     number to readers/UI; bearishness is in sentiment
    return {
        "ticker": row["ticker"],
        "sentiment": sentiment,
        "pc_ratio": pc,
        "call_premium": cp,
        "put_premium": pp,
        "total_premium": row["total_premium"],
        "net_premium": cp - pp,
        "call_volume": row["call_volume"],
        "put_volume": row["put_volume"],
        "change_pct": row.get("change_pct"),   # None — snapshot dropped in Chunk 4 (§7b)
        "source": "railway_poller",
        "updated_at": datetime.now(timezone.utc).isoformat(),
    }
```
