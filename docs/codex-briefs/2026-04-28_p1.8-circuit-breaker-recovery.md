# P1.8 — UW Circuit Breaker Recovery Hotfix

**Type:** Hotfix (production incident)

## Context

The UW circuit breaker in `backend/integrations/uw_api.py` has a stale-state bug: once tripped, it can mathematically never recover on its own.

`_circuit_breaker_open()` checks if the cooldown window has expired, but **never resets `_cb_failures` back to 0** when it does. So after the 5-minute cooldown ends, the failure count is still >= 5 (currently sitting at 147 in production). The next single failure increments to 148, which is still >= `CB_FAILURE_THRESHOLD` (5), so `_record_failure()` immediately re-opens the breaker for another 5 minutes. Loop forever.

Verified upstream is healthy:
- `UW_API_KEY` on Railway matches the canonical key in `claude_desktop_config.json`
- That same key successfully calls `https://api.unusualwhales.com/api/stock/XLK/state` from the UW MCP server right now (returned XLK $158.30, prev close $157.85)
- The original failure spike (whatever tripped the breaker hours ago) has long since passed — P1.5/P1.6/P1.7 fixes already shipped today address it. The breaker is just stuck.

The 2.7% cache hit rate is a *symptom*, not a separate bug — when the breaker is open, `_uw_request` returns `None` without ever reaching `cache_set`, so nothing populates. It will refill naturally once the breaker stops blocking writes.

Patch `_circuit_breaker_open()` to reset breaker state when the cooldown window has elapsed. Redeploy. Done.

## The Change

**File:** `backend/integrations/uw_api.py`

**Find (exact match — currently lines ~50-54):**

```python
def _circuit_breaker_open() -> bool:
    """Check if circuit breaker is open (should NOT make requests)."""
    if _cb_open_until > 0 and time.time() < _cb_open_until:
        return True
    return False
```

**Replace with:**

```python
def _circuit_breaker_open() -> bool:
    """Check if circuit breaker is open (should NOT make requests).

    P1.8 fix 2026-04-28: Reset breaker state once cooldown elapses. Previously
    `_cb_failures` was never zeroed when the cooldown window ended, so the stale
    count (often >>5) caused the very next failure to immediately re-trip the
    breaker. Effect: once tripped, breaker could only recover via process restart.
    """
    global _cb_failures, _cb_open_until
    if _cb_open_until > 0 and time.time() < _cb_open_until:
        return True
    # Cooldown expired (or never opened) -- ensure clean state for next attempt window
    if _cb_open_until > 0:
        _cb_failures = 0
        _cb_open_until = 0.0
    return False
```

That's the only change. Do not modify `_record_success()`, `_record_failure()`, or any constants.
